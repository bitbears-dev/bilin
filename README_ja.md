# Bilin

> **[Read in English 🇬🇧/🇺🇸](./README.md)**

Bilin は、英語で書かれた長文の海外ニュースやブログ記事を、語学学習も兼ねてスムーズに読み進めるために開発された AI 搭載型のデスクトップアプリケーションです。
オリジナルの Web サイトと、段落ごとに抽出された対訳カードを並列（Side-by-Side）で表示でき、読み上げ機能にも対応しています。

## 🚀 主な機能

- **並列表示 (Side-by-Side) レイアウト**: 中央ペインに本物の Web サイトのオリジナルビューを残したまま、右ペインの抽出されたカードで学習を進めることができます。
- **段落ごとの AI 翻訳**: OpenAI, Gemini, Claude などの任意の大規模言語モデルを活用し、文脈に沿った自然な日本語訳をパラグラフ単位で必要な分だけ表示します。
- **ネイティブ音声の読み上げ (TTS)**: 各段落の「Listen」ボタンを押すことで、OpenAI や Gemini を用いて指定した英語の自然な読み上げ音声を生成し、リスニング学習に活用できます。
- **ローカルキャッシュでの節約**: 一度翻訳したテキストや取得した音声データは、自動的に IndexedDB に保存されます。再度学習する際の API 呼び出しコストや待機時間はゼロになります。
- **複数タブ (セッション) 管理**: アプリを閉じても状態が維持されるセッション一覧に対応。各タブのスクロール位置も含めて記憶するため、「あとで読む」用途でも途切れることなく学習を再開できます。
- **BYOK (Bring Your Own Key) 方式**: 翻訳や音声生成に必要な API キーはご自身のものを入力頂きます。キーはデバイスのローカルストレージにのみ保存され、安全です。

## 📦 インストール方法

現在の環境に導入するための主な2つの手段をご用意しています。

### Option 1: ビルド済みアプリのダウンロード（推奨）
本アプリは GitHub Actions を用いて、リリース毎に各 OS 向けの実行ファイルを自動で生成しています。
1. プロジェクトの [Releases タブ](../../releases) へアクセスします。
2. お使いの OS に合わせたファイルを選んでダウンロードします。
   - **Windows**: `.msi` または `.exe` ファイル
   - **macOS**: `.dmg` ファイル
   - **Linux**: `.AppImage` または `.deb` パッケージ
3. ダウンロードしたファイルを開いてインストールを実行します。

### Option 2: ソースからのビルド（開発者向け）
ご自身の環境でソースコードから直接アプリをビルドする場合の手順です。実行には、**Node.js** および **Rust Toolchain** のインストール環境が必要です。

```bash
# 1. リポジトリをクローンする
git clone https://github.com/bitbears-dev/bilin.git
cd bilin

# 2. 依存パッケージのインストール
npm install

# 3. 開発用サーバー (Tauri) でアプリを起動
npm run tauri dev

# 4. (任意) お使いのOS用のアプリ実行バイナリをビルド
npm run tauri build
```

## 📖 基本的な使い方

1. **APIキーとモデルの設定**: 画面の左下にある **設定（歯車アイコン）** をクリックします。
   - 翻訳用のプロバイダー（OpenAI, Gemini, Claude）を選び、APIキーを入力してモデルを選択します。
   - 音声 (TTS) 用のプロバイダー（OpenAI, Gemini）を選び、APIキーを入力してモデルを選択し、「保存」します。
2. **記事を開く**: 画面上部の URL 入力バーに、読みたい英語のニュースや記事等のリンクを貼り付け、**「開く」** を押します。
3. **読んで学ぶ**: 記事の本文が抽出され、右側のパネルにパラグラフごとのカードになって表示されます。
   - 意味がわからないところは **「翻訳」** をクリックします。
   - 発音を確認したいときは **「Listen」** をクリックします。音声はいつでも **一時停止 (Pause)** や **停止 (Stop)** が可能です。
   - 右ペイン上部にある **「自動翻訳 (Auto Translate)」** トグルをオンにすると、スクロールしてカードが画面内に入ったタイミングで自動的に翻訳が行われます。
   - 中央ペインのオリジナル記事内にある段落の文章を直接クリックすると、右側のパネルが自動でスクロールし、対応するカードにフォーカスされます。
4. **タブの管理**: 別のサイトも見たい場合は、左側の垂直タブにある **「＋」アイコン** を押すことで、現在の状態を保持したまま新しい空のタブを作成できます。タブを切り替えても、各記事のスクロール位置はスムーズに復元されます。

## 🚢 リリース手順

新しいバージョンの Bilin をリリースする際は、npm の組み込みバージョン管理システムを利用します。これにより、必要なすべてのファイルのバージョン番号を一括更新し、GitHub Actions 用のタグを自動で発行できます。

1. 作業ディレクトリがクリーンな状態（未コミットの変更がない状態）であることを確認します。
2. 以下のコマンドを実行してバージョンを上げます（例：`1.2.3` の場合）：
   ```bash
   npm version 1.2.3
   ```
   *このコマンドにより、裏側で以下の処理が全自動で行われます：*
   - *`package.json` と `package-lock.json` の更新*
   - *`src-tauri/Cargo.toml` と `src-tauri/tauri.conf.json` の更新*
   - *上記の変更の Git コミットと、`v1.2.3` という Git タグの作成*
3. 作成されたコミットとタグをリモートリポジトリにプッシュします：
   ```bash
   git push origin main --tags
   ```
4. GitHub Actions のリリースワークフローが自動的に起動し、Windows, macOS, Linux 向けのバイナリをビルドして、Releases ページに新しいドラフトリリースを作成します。
